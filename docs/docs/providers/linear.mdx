# Linear

## Overview

**Category:** `ticketing`

| Feature                              | Available      |
| ------------------------------------ | -------------- |
| Authentication (`oauth2`)            | Yes            |
| Managed syncs                        | No             |
| &nbsp;&nbsp;&nbsp; Sync strategies   | (listed below) |
| Unified API                          | Yes            |
| &nbsp;&nbsp;&nbsp; Data invalidation | No             |
| Real-time events                     | No             |
| Passthrough API                      | Yes            |

#### Supported common objects:

N/A

#### Supported standard objects:

| Object     | Soft delete supported | Sync strategy       |
| ---------- | --------------------- | ------------------- |
| `issues`   | No\*                  | Full or Incremental |
| `comments` | No\*                  | Full or Incremental |
| `teams`    | No\*                  | Full or Incremental |
| `projects` | No\*                  | Full or Incremental |
| `users`    | No\*                  | Full or Incremental |

[*] Soft deletes are supported if the sync strategy is "Full"

#### Supported custom objects:

N/A

## Provider setup

Scopes: `read`, `write`, `issues:create`, `comments:create`. (Alternatively, set `admin` if you want full access to everything.)
